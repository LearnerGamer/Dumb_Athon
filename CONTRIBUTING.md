# Contributing to BUNKR

First off, thank you for considering contributing to BUNKR! It's people like you that make this simulated apocalypse an exciting place to exist.

## Where to Start?
If you're looking for a way to contribute, check our issue tracker for anything marked "good first issue" or "help wanted."
If you have a feature idea, please open an issue first to discuss it so we can align it with the project's vibe.

## Code Style
This project is deliberately stylized to be chaotic on the front end (especially within the `BUNKR` directory), but we aim for clean, maintainable code on the backend and in the Next.js app router structure.

When writing React/Next.js code:
- Prettier and ESLint are your friends. 
- Try to use TypeScript whenever possible.
- Keep components modular, even if they render chaotic, horrifying elements.

When editing `BUNKR/index.html`:
- Embrace the chaos, but keep things readable for other developers. 
- Inline styles are allowed here for authenticity, but CSS within the `<style>` block is preferred.
- Old-school HTML (like `<marquee>` or tables) is highly encouraged in this specific file to maintain the 90s aesthetic.

## Pull Request Process
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. If you've changed APIs, update the documentation.
4. Ensure the test suite passes (if available).
5. Make sure your code lints.
6. Issue that pull request!

## Any questions?
Feel free to ask questions in the issues.
